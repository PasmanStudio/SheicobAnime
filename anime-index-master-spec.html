Section 01

Architecture overview

The platform is a modular monolith — not microservices. Four independently deployable concerns, all sharing a codebase, designed to decompose at clean interfaces when traffic demands it.

Core design principle

The platform never stores video. It indexes publicly embeddable <iframe> URLs from third-party mirrors, validates they are embeddable by checking X-Frame-Options and CSP headers, stores them with a priority score, and serves them to the frontend which handles sequential fallback in the browser. This positions the platform legally as a search index, not a host.

Scaling contract #1 — cache portability
ICacheService is the only interface the API calls. Phase 1: Upstash HTTP. Phase 3: Redis Cluster. One DI line changes. Zero other code changes.
Scaling contract #2 — scraper portability
IScrapeStrategy is the only interface the scheduler calls. Each strategy is independently containerizable. Adding a source = one new class.
Scaling contract #3 — ad portability
<AdSlot placement="..."> is the only call site across all pages. Swapping ad networks = changing one component file. All pages unchanged.
Scaling contract #4 — comment portability
COMMENT_PROVIDER env var switches between Disqus, Remark42, and native. The episode page layout never changes.

Database schema

Designed for read-heavy workloads. series and episodes are the hot tables. Upsert-safe via ON CONFLICT DO UPDATE. Full-text search via generated tsvector column — swapped to Typesense in Phase 3.

schema.sqlPostgreSQL 16
-- Core tables. Run via EF Core migrations (dotnet ef database update).

CREATE TABLE series (
  id              UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  slug            TEXT        UNIQUE NOT NULL,
  title           TEXT        NOT NULL,
  title_romaji    TEXT,
  title_native    TEXT,
  synopsis        TEXT,
  cover_url       TEXT,          -- stored as path; served via STORAGE_BASE_URL env var
  banner_url      TEXT,
  year            SMALLINT,
  status          TEXT        CHECK (status IN ('ongoing','completed','upcoming','hiatus')),
  type            TEXT        CHECK (type IN ('tv','movie','ova','ona','special')),

PlaywrightScraperBase.csC# — shared browser infrastructure
public abstract class PlaywrightScraperBase : IScrapeStrategy, IAsyncDisposable
{
    private static readonly string[] UserAgents = [
        "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 Chrome/122.0.0.0 Safari/537.36",
        // ... 19 more real browser UAs rotated per request
    ];

    protected async Task<IPage> GetPageAsync()
    {
        var browser = await _playwright.Chromium.LaunchAsync(new() {
            Headless = true,
            Args = ["--no-sandbox", "--disable-gpu", "--disable-dev-shm-usage"]
        });

        var ua = UserAgents[Random.Shared.Next(UserAgents.Length)];
        var page = await browser.NewPageAsync(new() { UserAgent = ua });

        // Intercept ALL network requests — captures dynamically injected iframe src values
        // that HTTP-only scrapers completely miss.
        page.RequestFinished += (_, req) => {
            if (req.Url.Contains("embed") || req.Url.Contains("player"))
                _capturedEmbeds.Add(req.Url);
        };
        return page;
    }

    protected async Task JitterDelayAsync()
        => await Task.Delay(Random.Shared.Next(2000, 5000)); // 2–5s random delay
}
MirrorProbeService.csC# — embeddability gate
// Only indexes URLs the host has explicitly made embeddable.
// This is the legal and technical gate — rejects any URL that blocks iframes.
public async Task<bool> IsEmbeddableAsync(string url)
{
    var response = await _client.SendAsync(
        new HttpRequestMessage(HttpMethod.Head, url));

    if (!response.IsSuccessStatusCode) return false;

    if (response.Headers.TryGetValues("X-Frame-Options", out var xfo))
        if (xfo.First().ToUpper() is "DENY" or "SAMEORIGIN") return false;

    if (response.Headers.TryGetValues("Content-Security-Policy", out var csp))
        if (csp.First().Contains("frame-ancestors 'none'")) return false;

    return true;
}

Scheduler strategy

All jobs run via Hangfire with PostgreSQL backing. No additional infrastructure. Dead-letter jobs trigger a Resend email alert.

Phase 3 — queue upgrade path

Job method signatures are the contract. In Phase 3, swap Hangfire's PostgreSQL backing for MassTransit + RabbitMQ — register the same job classes as MassTransit consumers. The scheduler calls IJobRunner.EnqueueAsync<T>(), not Hangfire directly, so the swap is one implementation change.

Section 09

Mirror fallback logic

All fallback runs client-side. The API returns an ordered array of mirrors. The browser tries them sequentially with a 5-second timeout. Failures are reported back to the API asynchronously.

useMirrorFallback.tsTypeScript — client-side hook
const LOAD_TIMEOUT_MS = 5000;
const MAX_ATTEMPTS = 3;

export function useMirrorFallback(mirrors: Mirror[]) {
  const [index, setIndex] = useState(0);
  const [status, setStatus] = useState<'loading' | 'playing' | 'error'>('loading');
  const iframeRef = useRef<HTMLIFrameElement>();
  const attempts = useRef(0);

  const tryNext = useCallback((from: number) => {
    const next = from + 1;
    if (next >= mirrors.length || attempts.current >= MAX_ATTEMPTS) {
      setStatus('error'); return;
    }
    attempts.current++;
    setStatus('loading');
    setIndex(next);
  }, [mirrors.length]);

  useEffect(() => {
    const timer = setTimeout(() => {
      reportMirrorFailure(mirrors[index].id); // fire-and-forget
      tryNext(index);
    }, LOAD_TIMEOUT_MS);

    iframeRef.current?.addEventListener('load', () => {
      clearTimeout(timer); setStatus('playing');
    });

    return () => clearTimeout(timer);
  }, [index]);

  return { currentMirror: mirrors[index], status, iframeRef, selectMirror: setIndex };
}

function reportMirrorFailure(id: string) {
  // Never await — non-critical path, never block the UI
  fetch(`/api/mirrors/${id}/report`, { method: 'PATCH' }).catch(() => {});
}

Mirror priority formula

When the scraper stores mirrors, it calculates a priority integer (lower = tried first):

priority = round(
  (1 - quality_label / 1080) * 40   -- quality: 1080p gets priority 0, 360p gets 40
  + consecutive_failures * 10        -- failures push mirror down the list
  + provider_reputation_penalty      -- configurable per-provider offset
)

// ALL API calls flow through this module.
// NEXT_PUBLIC_API_URL is the ONLY thing that changes when the API scales.
// Phase 3: this URL becomes a Cloudflare-fronted load balancer. Zero frontend changes.

const BASE = process.env.NEXT_PUBLIC_API_URL;

async function apiFetch<T>(path: string, init?: RequestInit): Promise<T> {
  const res = await fetch(`${BASE}${path}`, {
    ...init,
    headers: { 'Content-Type': 'application/json', ...init?.headers },
    next: init?.next,   // passthrough for Next.js ISR cache hints
  });
  if (!res.ok) throw new ApiError(res.status, (await res.json()).error);
  return res.json();
}

// This component is NEVER included in SSR output.
// Search engines see the episode metadata (title, synopsis, thumbnail)

Ads integration

AdSlot is the only component that contains ad network code. Every page uses <AdSlot placement="...">. Swapping from AdSense to Ezoic to Mediavine requires changing one file.

AdSlot.tsxReact — ad network replacement contract
'use client';
// SCALING CONTRACT: This is the ONLY file with ad network code.
// AD_CONFIG is the single source of truth for ad unit IDs.
// Swap networks: change AD_CONFIG + AdUnitComponent. All <AdSlot> call sites unchanged.

const AD_CONFIG = {
  'above-fold':      { slot: '1234567890', minHeight: 90  },
  'below-player':   { slot: '2345678901', minHeight: 250 },
  'below-episodes': { slot: '3456789012', minHeight: 250 },
} as const;

export function AdSlot({ placement }) {
  const ref = useRef();
  useEffect(() => {
    const obs = new IntersectionObserver(([e]) => {
      if (e.isIntersecting) {
        // push() the ad unit lazily — only when the slot is in viewport
        ((window as any).adsbygoogle ??= []).push({});
        obs.disconnect();
      }
    });
    if (ref.current) obs.observe(ref.current);
    return () => obs.disconnect();
  }, []);

  return (
    // min-height prevents CLS before ad loads — required for Core Web Vitals
    <div ref={ref} style={{ minHeight: AD_CONFIG[placement].minHeight }}>
      <ins className="adsbygoogle"
           data-ad-client={process.env.NEXT_PUBLIC_ADSENSE_ID}
           data-ad-slot={AD_CONFIG[placement].slot}
           data-full-width-responsive="true" />
    </div>
  );
}

Comments integration

CommentSection.tsxReact — comment provider swap contract
'use client';
// COMMENT_PROVIDER env var controls the implementation.
// Phase 1: 'disqus' (free, instant)
// Phase 2: 'remark42' (self-hosted Go binary, ~30MB RAM on Hetzner)
// Phase 3: 'native' (GET /episodes/{id}/comments from your own API)
// Episode page layout never changes when switching providers.

export function CommentSection({ episodeId }) {
  const [visible, setVisible] = useState(false);
  const ref = useRef();

  useEffect(() => {
    const obs = new IntersectionObserver(([e]) => {
      if (e.isIntersecting) { setVisible(true); obs.disconnect(); }
    }, { rootMargin: '200px' }); // preload 200px before visible
    if (ref.current) obs.observe(ref.current);
    return () => obs.disconnect();
  }, []);

  const provider = process.env.NEXT_PUBLIC_COMMENT_PROVIDER ?? 'disqus';

  return (
    <section ref={ref}>
      {visible && provider === 'disqus'   && <DisqusEmbed   episodeId={episodeId} />}
      {visible && provider === 'remark42' && <Remark42Embed episodeId={episodeId} />}
      {visible && provider === 'native'   && <NativeComments episodeId={episodeId} />}
      {!visible && <div className="h-48 animate-pulse bg-gray-900 rounded-lg" />}
    </section>
  );
}

Section 14

Security considerations

API security
Admin endpoints require X-Admin-Key header. Rate limiting via ASP.NET RateLimiter middleware. CORS whitelist — no *. FluentValidation on all inputs. Max search query: 100 chars.
iframe sandboxing
All <iframe> embeds use sandbox="allow-scripts allow-same-origin allow-presentation allow-fullscreen". Restricts what the embedded player can do — no popups, no top-level navigation.
Content Security Policy
frame-src * is intentional (must allow arbitrary mirror domains). All other directives are strict. script-src whitelists only AdSense and Disqus domains.
Secrets management
Zero secrets in source code. All via Railway/Vercel environment variables. .env.example has keys only, never values. .env in .gitignore.
Section 15

Legal architecture

The architecture is designed around the indexer defense — the same legal framing used by search engines. Every technical decision reinforces this position.

Core legal position

The platform is a search index, not a host. Video is served from third-party hosts. Your servers store only metadata and URLs. The MirrorProbeService only indexes URLs the host has explicitly made embeddable — you are not circumventing any access control.

Content typeLegal riskMitigation
Metadata (title, synopsis)LowFactual data has weak copyright protection. Transformative use argument applies.
Cover images / thumbnailsMediumHost on R2 with attribution. Respond to takedowns within 24 hours.
Embed URLsMedium-HighDMCA safe harbor + blocked_slugs table + 24h takedown SLA
Actual video filesHighArchitecture explicitly prevents storing video. MirrorProbeService is the gate.

DMCA safe harbor checklist

  • Register a DMCA agent with the US Copyright Office ($6 one-time fee)
  • Publish a DMCA/takedown page at /dmca with contact email
  • DELETE /admin/series/{slug} endpoint: deletes series + adds to blocked_slugs
  • Scraper checks blocked_slugs before re-indexing any content
  • Implement repeat-infringer policy: ban IPs/sources that repeatedly re-add blocked content
  • Respond to all takedown requests within 24 hours (Resend email alert when received)

Section 17

Scaling roadmap

Three phases. No rewrites. Each upgrade is an infrastructure-level change or a DI registration swap. The codebase is identical across all three phases.

Phase 1 — Zero budget
0 → 10,000 MAU. Railway + Vercel + Supabase + Upstash. Prove the concept. Accumulate content. Reach AdSense approval.
Phase 2 — $20–50/mo
10k → 200k MAU. Hetzner VPS for API + dedicated scraper. Ezoic for ads. Remark42 for comments. Same codebase, new infra.
Phase 3 — Cloud-native
200k+ MAU. K8s or Fly.io replicas. PG read replica. Redis Cluster. Typesense search. Distributed scraper fleet.

The no-rewrite guarantee

Every scaling move is either (a) an infrastructure-level change with no code impact, or (b) swapping a DI-registered implementation behind an existing interface. The monolith codebase is deployed identically in all three phases. "Scaling" means decomposing the infra topology, not the application code. The four scaling contracts (ICacheService, IScrapeStrategy, AdSlot, COMMENT_PROVIDER) were designed on day one specifically to make this true.
